1 Objectif

L'objectif de cet exercice est d'observer les requêtes et réponses HTTP au cours de la navigation dans un navigateur Web.

2 Accès à l'inspecteur réseau

  1. Activer l'accès aux outils du développeur Web si nécessaire (Chrome)

  2. Accès aux "Outils de développement" dans Firefox ou Chrome, et à l'onglet "réseau".

    Note: un raccourci clavier pour l'accès direct est configuré sur Ctrl + MAJ + Q

  3. Configurer l'outil de développement pour garder une trace persistante de toutes les requêtes et réponses :
    • Dans les préférences des outils du développeur
      • Préférences générales
        • Activer les journaux persistants

3 Naviguer sur des sites et observer les requêtes et réponses

  • http://perdu.com/
  • http://www.telecom-sudparis.eu/
  • …

4 Navigation privée

Expérimentez la consultation dans le mode "navigation privée".

5 Observer le rôle du cache du navigateur

  1. Ouvrir http://perdu.com/ dans Firefox, une fois la vue de l'inspecteur réseau ouverte dans les outils du développeur.

    Le code de réponse est 200 la première fois

  2. Recharger la page (F5 ou Ctrl + R ou icône d'actualisation dans la barre d'URL).

    Cette fois, le code de réponse est 304. Expliquer…

  3. Forcer le rechargement de la page : Ctrl + MAJ + actualisation (F5 ou Ctrl+R ou icône).

    Cette fois, le code de réponse est 200. Notez la couleur de l'icône dans l'inspecteur (rond plein vert).

  4. Recommencez la navigation en retapant la même URL (Ctrl+L pour se positionner dans barre d'adresse / URL) http://perdu.com/

    Cette fois-ci, la requête est refaite (code 200) mais l'icône est un rond vide. Expliquer…
